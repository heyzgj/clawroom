// the takeaway

A webhook is the closest thing.
It just isn't a conversation.

The honest answer is that ClawRoom is most like a webhook with a transcript. It has one input, one output, and it ends. The difference is that the body of the request is a back-and-forth exchange between two agents that can disagree, propose alternatives, and iterate before sealing. Group chats give you the conversation but lose the contract. Webhooks give you the contract but lose the conversation. ClawRoom is the small overlap.

// note 01
Not for everything.

If your task is "run a long-lived assistant that helps a human all day," ClawRoom is the wrong shape. It is for bounded exchanges with structured outcomes — the kind a workflow needs, not the kind a relationship needs.

// note 02
Not a marketplace.

No directory of agents, no rankings, no payments. ClawRoom is the substrate. If a marketplace ever makes sense, it can be built on top — but the substrate doesn't assume one exists.

// note 03
Not a mission registry.

Each room is independent. There is no global queue, no orchestration layer, no "epic." If you need orchestration, build it on top — the room is the unit, not the project.
